\section{Repository and runtime setup}
Use Python 3.10--3.13, install the pinned editable project, run Ruff and pytest, then execute the
smoke configuration. Every run stores the resolved YAML, source commit, package versions, device,
command, status, seed allocations, and artifact paths. A failed run retains its traceback and an
emergency checkpoint when possible.

\section{Detailed environment mechanics}
The field is $9\,\mathrm{m}\times6\,\mathrm{m}$ with a $1.5\,\mathrm{m}$ goal. A macro decision is
repeated for four $0.1\,\mathrm{s}$ low-level steps. The abstract model limits acceleration,
translation, and turn rate; applies linear ball drag; resolves circular contacts; and delivers kick
velocity only inside range and angle tolerances. The transfer model independently creates Pymunk
bodies, force controllers, collision shapes, walls with goal openings, damping, friction,
elasticity, and impulse kicks over multiple integration substeps.

Observation and action delay queues are bounded. Object misses set confidence to zero without
changing truth. Packet loss preserves the previous delivered teammate action and increments its
age. The intercept defender protects goal center for a distant ball, tracks a delayed predicted
intercept when threatened, and clears only inside control range.

\section{Perturbation-profile definitions}
Named profiles cover low/medium/high action delay; low/medium/high observation corruption;
low/medium/high communication impairment; weak/strong kicks; low/high ball friction; slow/fast
movement response; and combined moderate/severe conditions. Reset samples continuous values inside
each YAML range and records both profile and realization. Family-level ablations filter names rather
than changing simulator code.

\section{Curriculum-update derivation}
For success $q_i$, failure $1-q_i$ enters an EMA
$f_i\leftarrow d f_i+(1-d)(1-q_i)$. Weights $(\epsilon+f_i)^\alpha$ are normalized, mixed with
$\beta/N$, then projected onto the capped simplex. Defaults are $d=0.5$, $\epsilon=0.05$,
$\alpha=1$, $\beta=0.1$, and cap 0.25. Curriculum seeds begin at 20,000 and do not update running
normalizers.

\section{PPO implementation notes}
Rollouts are stored as time, environment, agent, feature arrays. MAPPO has one value and one GAE
advantage per environment transition; actor training broadcasts that advantage, while critic
training does not duplicate it. IPPO flattens local value samples across both agents. Truncations
bootstrap from the final observation but stop the recursive advantage chain across reset. Loss and
gradient finiteness is mandatory; a NaN raises after an emergency checkpoint attempt.

\section{Seed allocation}
Training begins at the experiment seed. Ordinary validation, curriculum validation, abstract final
test, Pymunk transfer, and video scenarios begin at 10,000, 20,000, 30,000, 40,000, and 50,000
respectively. Final runs must record every exact seed and must not reuse model-selection outcomes as
final tests.

\section{Discarded ideas}
Version one intentionally excludes image observations, joint-level locomotion learning, full 5v5
play, self-play, learned opponents, RNNs, ROS, and physical-deployment claims. Each would add an
independent source of variance or infrastructure before the cross-simulator causal comparison is
established.

\section{Webots setup attempts}
Webots is documented but neither scaffolded nor claimed as tested. Log simulator version, official
NAO/OP3 asset provenance, headless command, motion assets, controller trace, and exact failure here
before changing that status.

\section{Known limitations}
Skill adapters encode task knowledge; the defender is scripted; synchronous collection may limit
throughput; feed-forward actors infer delay only through short observation history; Pymunk remains
a simple planar world; and robustness profiles cannot span unmodeled hardware failures.

\section{Future physical-robot deployment plan}
First validate TorchScript observation parity in a supervised simulator. Then integrate a tested
localization and ball/robot perception stack, collision-safe walking and turning, controlled kick
motions, communication timestamps, action watchdogs, emergency stops, and conservative field
tests. A successful deployment requires logged physical trials, not merely controller startup.

\section{Reproducibility checklist}
\begin{itemize}
\item Record clean commit, resolved YAML, packages, device, and all seed ranges.
\item Pass Ruff, the full test suite, both PettingZoo API checks, and the smoke command.
\item Preserve complete checkpoints, normalizers, curriculum history, CSV metrics, and logs.
\item Evaluate frozen checkpoints on fixed abstract and Pymunk seeds without retraining.
\item Aggregate multiple seeds and report variability, tail metrics, and failed runs.
\item Rebuild both reports from restored Drive artifacts in a fresh Colab runtime.
\item Describe Pymunk as sim-to-sim and do not infer physical-robot performance.
\end{itemize}
